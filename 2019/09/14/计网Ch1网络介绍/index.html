计算机网络之网络介绍

Posted on 2019-09-14

1 什么是网络

  • 什么是网络

    • 端系统(host) 包括手机, PC…
    • 基站/访问点(access point)
    • 交换机(switch)
      • 路由器(router)特指运行IP协议的交换机
    • 通信链路: 光纤(fiber), 铜缆(Copper), 无线电(Radio), 卫星通信(Satellite), 双绞线(使用的以太网)

  • 构建网络 – 服务视角

    • Communication infrastructure(下部构造)
      • Enables distributed applications应用
      • Web, VoIP, email, on-line games, e-commerce, file sharing
    • 通讯服务提供
      • 高效但不可靠的数据传输 – IP
      • 从源到目的地的可靠数据传输 – TCP
      • 可保障的延迟和吞吐量 – QoS
    • 协议架构 Network Protocols
      • 控制报文的发送和接收
      • e.g. HTTP, Skype, TCP, IP, PPP, Ethernet
    • 网络标准
      • IETF: Internet Engineering Task Force
      • RFC: Request for comments
    • 互联网: 网络的网络
      • 私有网络和公共网络
      • 大致上阶级化

2 网络边缘

  • 网络边缘:
    • 端系统End system(主机hosts): 运行应用程序: web, e-mail
    • 客户/服务器模型 Client/server model
      • 上行(客户端向服务器发出请求)
      • 下行(服务器给客户端提供服务), 有瓶颈
    • Peer-to-peer model
      • 少用复杂的服务器, 如Skype, BitTorrent

2.1 接入网

  • 接入网 Access Network
    • 连接端系统和边缘路由器

  1. 家庭接入Residential Access:

    • 拨号上网Dialup via modem
      • 类似于DSL, 速度慢
    • 数字用户线接入 DSL: digital subscriber line
      • 使用现有的电话线(双绞铜线)
    • 电缆英特网接入: HFC: hybrid fiber coax 混合光纤同轴电缆
      • 利用有线电视公司的基础设施
      • 不对称: 下载快, 上传慢
      • 家庭共享接入ISP路由器

  2. 企业接入 company access

    • 局域网LAN连接端系统到端路由器
    • 以太网Ethernet

  3. 无线接入

    • 共享无限媒体连接端系统到路由器
      • 通过基站或者访问点
    • 无线局域网(Wireless LANs)
      • WiFi
    • 广域无线接入
      • LTE/WiMAX(4G)
  • 现代家庭网络包括
    • DSL或者电缆调制解调器
    • 路由器/防火墙/NAT
    • 以太网交换器/无限访问点

2.2 物理媒体

  • 分类:

    • 导引型媒体
      • 电波沿固体媒体前行
    • 非导引型媒体
      • 电波在空气或外层空间中传播

  • 双绞铜线

  • 同轴电缆

  • 光纤

  • 陆地无线电信道

  • 卫星无线电信道

3 网络核心

  • The Network core
    • 分组交换机和链路构成的网状网络, Mesh of interconnected routers
    • 核心问题:
      • 数据如何通过网络传播
    • 分类
      • 电路交换(面向连接, 固定的)
      • 分组交换(面向无连接, 到了每个路口自己决定怎么走)

3.1 分组交换

  • 类似于公路网, 或者打的
  • 源将报文(message)划分为较小的数据块, 称为分组(packet). 在源和目的地之间, 每个分组都通过通信链路和分组交换机传送
  • 无固定路线
  • 应用A, B的分组共享网络资源
  • 不分配资源
  • 分组每次前进一步虚电路: 后在交换机处存储(排队)
  • 每个包用满的带宽
  • 有拥塞问题, 排队

3.2 电路交换

  • 类似于铁路网

  • 端到端的资源为”call”保留, 专用资源, 无共享

  • 面向连接, 固定路线

  • 路由器分配资源: 带宽固定

  • 线路分配资源: 频分复用/时分复用/码分复用

  • 性能有保障

  • 有Call setup和teardown, 不灵活

3.3 虚电路

  • 分组交换 + 电路交换, 分组交换和电路交换的中间形式(类似于公交车)
    • 路由器或者主要通路固定
    • 资源共享, 需要拥塞控制
    • 资源可被保留, 导致不同的性能
    • 需要连接setup/teardown

3.4 网络的网络

  • 粗略分等级:
    • 中心(1级): 国际ISPs(网络服务提供者)
    • 2级ISPs: 较小的(一般地区性的)ISPs
    • 3级ISPs和局部/端ISPs, 连接接入网

4 传统网络应用

  • 体系结构分类:
    • 客户-服务器体系结构 Client/Server Applications
    • P2P体系结构

4.1 客户-服务器体系结构

  • 服务器:
    • 总是开着的主机
    • IP地址固定
    • Server farms for scaling
  • 客户端
    • 和服务器交流
    • 间歇连接
    • 可能动态IP
    • 不直接和其他客户端通信

4.1.1 Web和HTTP

  • 客户用浏览器通过HTTP发送URL给服务器, 请求网页
  • 网页用HTML建立, 用URL相互连接
  • 服务器(或cache)回应网页请求
  • 客户端浏览器显示返回的网页
  • 来自多个服务器的会话(session): 超链接
  • HTTP: 传输, HTML: 表示

4.1.2 FTP

  • 从/向远程主机传输文件

  • 控制连接

    • login/logout, 文件传输命令和回应

  • 数据连接

    • 传输文件内容
    • 客户端开始文件传输

4.1.3 电子邮件

  • MIME表示协议
    • 非文本数据表示
  • SMTP简单邮件传输协议
    • 传输简单文本邮件
  • POP邮箱访问协议
    • 从服务器取回邮件, 包括授权和下载
  • IMAP网络邮件访问协议
    • 操作服务器上存储的邮件

4.2 P2P体系结构

4.2.1 BitTorrent(文件共享)

  • 文件文成256kB的文件块(chunks)

  • torrent: 一群交换文件块的peers

  • tracker: 追踪torrent中的peers

  • Peer(Alice)加入torrent:

    • 向tracker登记, 得到peers列表, 连接部分peers(邻居)
    • Alice发送缺失文件块的请求

  • 下载的同时, Alice向其他peers上传文件块

    • 同时最快速向4个邻居发送她的文件块

  • Peers可以加入或者退出

4.2.2 Skype(因特网电话和视频会议)

  • P2P Voice-Over-IP(VoIP)应用

    • pc-to-pc, pc-to-phone, phone-to-pc
    • 私人应用层协议
    • 结构: Skype clients(SC), Supernode(SN), Skype登录服务器

  • Skype: Makeing a call

    • alice打开Skype
    • SC先在supernode上注册
    • SC登录(鉴别)
    • SC把Bob地址发送给supernode
    • supernode联系其他supernode来找到bob的地址
    • alice直接连接bob, 通过UDP/TCP

5 协议层和服务器模型

  • 一个为文件传输应用的分层结构

  • 协议: 处理同层实体之间的通信问题

  • 三层的功能

    • 网络层
      • 将数据从源传到目的地
      • 源和目的地的地址
      • 通过网络路由数据
    • 传输层
      • 端主机之间的可靠数据传输
      • 错误修复, 流控制(防止传输太快)
    • 应用层
      • 处理不同主机之间的文件传输
      • 登录/登出; session控制; 格式转换

  • 地址需求

    • 至少两个地址
      • 网络地址用来标识主机
      • 标识应用
    • sockets套接字
    • 4级地址: 主机IP, 端口号, MAC地址, 应用标识

  • 标准协议体系

    • OSI(过时)
    • TCP/IP
    • Signaling System
    • …

5.1 OSI

  • Open Systems Interconnection

  • 由International Organization for Standardization (ISO)制定

  • 总共7层

  • 只是理论系统, 出现太晚

  • TCP/IP才是现在实际上的标准

  • 特点

    • 分层模型, 流结构
    • 每层是需要的通信功能的一个子集
    • 每层依赖下一层(更低的那层)来实现功能
    • 一层的变化不应该影响别的层

  • 物理层(搬运)

    • 通过物理介质传输比特
    • 详述:
      • 机械上: cable, plugs, pins
      • 电的/光学的
      • 功能/程序
    • 不同设备的物理界面
      • 以太网, DSL, 电缆调制解调器…
      • 双绞电缆, 同轴电缆, 光纤, 无线电, 红外线…

  • 数字链路层(装拆箱)

    • 把比特分组成帧(frames), 维持连接, 通过直接连接传递数据
    • multiple access则需要Medium access control(MAC)
    • 检错, 重传
    • 流控制
    • 更高层或许需要无错传输

  • 网络层(排序工人)

    • 通过多重连接/多重网络传输分组(packets)
    • 寻址必须扩展到大型网络
    • 结点执行路由算法, 决定网络路径
    • 推进分组穿过结点传播
    • 拥塞控制
    • 连接建立\保持\取消

  • 传输层

    • 数据在端系统直接交换
      • 端到端传输数据, 从一个主机的进程到另一个主机的进程
    • 可靠的流传递, 或者快而简单的单块传递
      • 无错, 依次, 无丢失, 无副本
    • 连接的建立, 保持和释放

  • 会话层, 表示层, 应用层

    • 现在总称为应用层
  • 服务早期
    • connect.request -> connect.indication -> connect.response -> connect.confirm

5.2 TCP/IP

  • 应用层(application): 支持网络应用
    • FTP, SMTP, HTTP
  • 传输层(transport): 进程间数据传输
    • TCP, UDP
  • 网络层(Internetwork): 将称为数据报(datagram)的网络层分组从一台主机移动到另一台主机
    • IP, routing protocols
  • 链路层(Link): 网络层通过源和目的地直接的一系列路由器路由数据报, 链路层分组为帧
    • PPP, Ethernet
  • 物理层(Physical): 将帧中的一个个比特从一个结点移动到另一个结点

5.3 协议数据单元 (Protocol Data Units PDU)

  • 在每一层, 为了便于通信, 控制信息被加入用户数据
  • 每一segment加上一个传输头
    • 目标SAP
    • 序列号
    • 检错码
  • 这就产生了传输协议数据单元(PDU)
